# 국시 플래너 메인 페이지 — Pretext-inspired 방향 설계 & 비서 자비스 인계

## 결정 사항
강렬 요청에 따라, 국시 준비 플래너의 **메인 페이지(first screen / landing / home)** 는
기존의 평범한 대시보드형 UI가 아니라,
**Pretext-inspired editorial / living typography 방향**으로 재설계한다.

단, 중요한 원칙은 다음과 같다.

> **메인 페이지는 시그니처 경험으로 과감하게 만들고,
> 실제 일정 입력/과목 상세/주간 플래너 화면은 계속 실무형으로 둔다.**

즉,
- 메인 페이지 = 감정/집중/방향성을 주는 "입구"
- 내부 페이지 = 읽기 쉽고 빠르게 수정되는 도구형 화면

---

## 왜 이 방향이 맞는가
강렬은 `pretext.cool`의 `Illustrated Manuscript` 계열 인터랙션에서
다음 매력을 확인했다.

1. 텍스트가 단순 정보가 아니라 **살아 움직이는 재료**처럼 보임
2. 강한 세계관이 있어 **간지가 남음**
3. 흔한 AI 대시보드 템플릿 느낌이 안 남

하지만 동시에,
플래너 전체를 그 스타일로 밀면 실사용성이 떨어진다.
따라서 Pretext는 **메인 페이지의 서사/무드/타이포그래피 엔진**으로만 쓰고,
실제 입력·편집 화면은 기능 중심으로 유지한다.

## 디자인 컨셉 이름
### 작업 컨셉
**Illuminated Study Codex**

### 컨셉 설명
- 중세 필사본/에디토리얼/학습 의식(ritual) 느낌
- 단순 판타지풍이 아니라 **공부를 시작하는 의식적 진입감**
- 국시 준비의 무게감, 집중감, 전략성을 시각화
- 드래곤/환상 요소를 그대로 복제하지 말고,
  **텍스트 흐름 / 장식 이니셜 / 주간 선언문 / 인장(sigil) 같은 요소**로 번역

### 톤
- 차분함
- 학구적임
- 의식적 집중감
- 약간의 장엄함
- 하지만 전체 사용성은 유지

## 메인 페이지가 해야 하는 일
메인 페이지는 단순히 보기 좋기만 한 화면이 아니라,
강렬이 앱에 들어왔을 때 아래 5가지를 즉시 전달해야 한다.

1. **이번 주 공부 모드가 무엇인지**
   - 신규 진도 중심인지
   - 복습 중심인지
   - 실습 대응 모드인지
2. **무엇이 가장 중요한지**
   - 최우선 과목 1~3개
   - 가장 밀린 단계
3. **오늘 실제로 쓸 수 있는 시간이 언제인지**
   - 전공의가 제공한 hard constraint 반영
4. **당장 눌러야 할 한 가지 행동**
   - 오늘 학습 시작
   - 주간 플랜 보기
   - 밀린 일정 정리
5. **공부 상태를 한 문장으로 느끼게 하기**
   - 예: "이번 주는 호흡기 신규 진도보다 순환기 복습을 먼저 지켜라."

---

## 메인 페이지 구조 제안
### Section 1 — Hero / Weekly Declaration
메인 페이지 최상단은 일반적인 숫자 카드가 아니라,
**이번 주 선언문(weekly declaration)** 이 중심이다.

예시 카피:
- "이번 주는 호흡기 신규 진도보다 순환기 복습을 먼저 지켜라."
- "실습이 빡센 주다. 유지 루프를 끊지 않는 것이 승리다."
- "고비중 × 저숙련 × 망각 큰 파트부터 다시 조인다."

표현 방식:
- 큰 타이포그래피
- 텍스트 reflow / line-shift / subtle reveal animation
- 배경 장식은 질감/문장 기호/도식 기반
- 숫자 KPI는 hero 밑의 보조 정보로 내림

### Section 2 — Mode Banner
이번 주 모드를 짧게 명시:
- 유지·복습 모드
- 신규 진도 확장 모드
- 실습 대응 모드
- 월말 총정리 모드

이 정보는 academic 로직에서 산출하고,
메인 페이지에서는 **강한 배지/리본/인장**처럼 표현한다.

### Section 3 — Priority Triad
이번 주 핵심 우선순위 3개
- 과목
- 단계
- 이유(고비중/저숙련/망각 등)

예시:
- 순환기 · 복습 · 고비중 + 최근 망각 큼
- 호흡기 · 개념틀 · 실습 연동 직전
- 산과 · 문제풀이 · 회독 부족

### Section 4 — Today Windows
전공의가 준 현실 시간 제약을 반영해,
오늘 실제로 쓸 수 있는 시간창을 보여준다.

예:
- 07:20–08:10 가벼운 복습
- 19:30–21:10 깊은 집중 블록
- 21:20–22:00 오답정리 가능

이 영역은 시각적으로는 아름다워도,
정보 전달은 명확해야 한다.

### Section 5 — Single CTA
메인 페이지에 버튼이 너무 많으면 안 된다.
핵심 CTA는 1개만 강하게 둔다.

기본 CTA 후보:
- 오늘 학습 시작
- 오늘 플래너 열기
- 밀린 일정 정리 시작

보조 CTA는 최대 2개까지만 둔다.

## 절대 지킬 제약
### 1. 메인 페이지만 실험적
- 일정표/폼/과목 상세까지 Pretext 스타일로 덮지 않는다.
- 실무 화면은 가독성 우선.

### 2. 텍스트는 여전히 읽혀야 함
- 예술 실험이 아니라 플래너 메인 화면이다.
- 멋있어도 읽기 힘들면 실패.

### 3. 장식은 데이터보다 앞서면 안 됨
- 선언문과 분위기는 중요하지만,
  실제 우선순위·시간창·CTA는 항상 한눈에 보여야 한다.

### 4. 판타지 과몰입 금지
- 드래곤/룬/장식 문자에 취해 앱 정체성이 흔들리면 안 된다.
- 목표는 "중세 판타지 사이트"가 아니라
  **강한 세계관을 가진 국시 플래너의 입구**다.

## 시각 언어 가이드
### Typography
- 본문 전체를 실험하지 말고,
  hero와 선언문에만 강한 타이포그래피 사용
- display type + 읽기 쉬운 body 조합
- 한국어 가독성이 우선
- 줄 단위 reflow / reveal / justified composition을 고려

### Color
- pure black / pure white 피하기
- 양피지/잉크/청색 인장 느낌의 톤 가능
- 혹은 아주 밝은 paper surface + cool ink tone
- 강한 accent는 1개만

### Motion
- hero 진입 시 한 번 강하게
- 이후에는 잔잔한 상태 변화만
- 주간 선언문 / mode badge / CTA에만 집중 모션
- 실제 플래너 내부는 과한 모션 금지

### Layout
- 일반 카드 그리드보다
  **에디토리얼 composition** + 명확한 정보 블록 조합
- 좌우 비대칭 허용
- 하지만 CTA와 우선순위는 시선 흐름 안에 명확히 배치

## 데이터 연결 규칙
이 메인 페이지는 예쁜 껍데기가 아니라,
secretary가 내부 협업 결과를 요약해 보여주는 화면이어야 한다.

### 자비스 전공의(clerk)로부터 받는 값
- 이번 주 / 오늘 hard constraint
- 공부 불가 시간
- 가벼운 복습만 가능한 시간
- 피로도 높은 날

### 자비스 교수님(academic)로부터 받는 값
- 이번 주 우선 과목 3개
- 각 과목의 가장 밀린 단계
- 이번 주 모드(유지/신규/월말 총정리 등)
- 최소 복습 루프
- 주간 선언문 초안에 들어갈 핵심 전략 문장

### 플래너 앱 상태에서 받는 값
- 밀린 일정 수
- 오늘 일정 수
- 전체 진행도
- 과목별 진행도/취약 파트

## secretary의 역할 (이번 건에서)
비서 자비스는 이 메인 페이지 프로젝트에서 다음을 맡는다.

1. **메인 페이지에 들어갈 weekly declaration 문구 관리**
2. 전공의/교수님에게서 받은 정보를
   - hero 문장
   - mode 배지
   - priority triad
   - today windows
   형식으로 정리
3. 주간 계획이 바뀌면 메인 페이지 콘텐츠 스펙도 갱신
4. 코드 수정이 필요하면 메인 자비스에게 넘김

즉,
secretary는 이 메인 페이지의 **콘텐츠 오너 + 통합 PM**이다.

## 메인 자비스에게 넘길 구현 단위
secretary가 구현 요청할 때는 아래 단위로 나눠서 넘기는 것이 좋다.

### Phase 1 — Static concept
- 메인 페이지 정보 구조 확정
- 선언문 중심 hero
- mode banner
- priority triad
- today windows
- CTA

### Phase 2 — Motion & typography
- declaration reveal
- subtle text reflow / line transition
- background texture / sigil treatment

### Phase 3 — Real data binding
- secretary가 모은 실데이터 연결
- 전공의/교수님 결과를 메인 페이지에 자동 반영

### Phase 4 — polish
- 모바일 최적화
- reduced motion
- 성능 안정화

## 최종 한 줄 정리
비서 자비스는 앞으로 국시 플래너 메인 페이지를
**Pretext-inspired living typography 메인 화면**으로 고도화하는 프로젝트를 관리한다.
하지만 그 실험성은 메인 페이지에만 집중시키고,
실제 일정/입력/과목 관리 화면은 계속 도구형 UI로 유지해야 한다.
