## 현재 산출물
- 경로: `~/.openclaw/workspace/kmle-planner/index.html`
- 형태: 정적 로컬 웹앱
- 저장: 브라우저 localStorage
- 백업: JSON export/import 지원

## 현재 구현 범위
1. 과목별 진행도 관리
   - 강의 듣기
   - 개념 틀 짜기
   - 문제 풀기
   - 오답노트/개념틀 수정
   - 복습하기
2. 과목별 학습량 입력 (`done/total`)
3. 오늘 일정 / 이번 주 일정 / 밀린 일정 표시
4. 복습·오답정리 우선 블록 표시
5. 취약 파트 / 다음 액션 / 과목 메모 저장
6. 예시 데이터 포함

## 왜 자비스 교수님에게 넘기는가
이 플래너는 단순 일정앱이 아니라, 결국 다음 작업과 강하게 연결된다.

- 알렌/강의/팟캐스트 기반 학습량 반영
- 과목별 source pack 진행도 반영
- 문제풀이 및 오답노트 루프 반영
- 복습 주기 추천
- 국시 전체 우선순위 조정

즉 **국시 도메인 이해가 있는 봇**이 붙어야 진짜 완성도가 올라간다.
그 역할은 메인 자비스보다 `자비스 교수님(academic)`이 더 자연스럽다.

---

## 인수인계 범위
### 1단계 — 운영 인수
자비스 교수님이 해야 할 일:
- 이 플래너의 과목 구조를 실제 국시 과목 체계에 맞게 정리
- 과목별 기본 목표량 세팅
- 강의/문풀/오답/복습 기준 정의
- 강렬의 실제 학습 흐름에 맞게 예시 데이터를 실데이터로 교체

### 2단계 — 기능 인수
다음 기능을 자비스 교수님이 우선 설계/주도:
- 반복 복습 자동 생성
- 취약 파트 기반 다음 액션 추천
- 시험일까지 남은 시간 대비 과목 우선순위 계산
- 일정 과부하 감지 (하루 학습량 과다 시 경고)
- 과목별 완성도 score 산출

### 3단계 — 연동 인수
후속으로 고려:
- Obsidian `Schedule.md` 또는 국시 노트와 연결
- 팟캐스트/강의록 제작 진행도와 연결
- 오답노트 입력을 받아 자동으로 해당 과목 진행도 보정

## 권장 핸드오프 패킷
자비스 교수님에게 넘길 때 최소한 이 4가지는 같이 전달:

1. **앱 파일 경로**
   - `~/.openclaw/workspace/kmle-planner/index.html`
2. **현재 요구사항 요약**
   - 과목별 5단계 진행 관리
   - 학습량 체크
   - 일정/복습 루프 관리
3. **남은 TODO**
   - 실제 과목 구조 정리
   - 반복 복습 로직
   - Obsidian 연동 여부 판단
4. **운영 원칙**
   - 사용자는 복잡한 설정 없이 바로 쓰길 원함
   - 보이는 UI보다 실사용성과 지속성이 더 중요
   - 데이터는 잃지 않게 export/import 또는 파일 저장형 개선 필요

> 다음 단계로는 실제 국시 과목 구조 반영, 반복 복습 자동화, 오답노트/강의록 흐름과의 연결이 필요합니다.  
> 이 플래너를 academic 관점에서 실사용형으로 고도화해 주세요.
