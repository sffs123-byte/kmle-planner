# 국시 준비 플래너 → 비서 자비스 인수인계 메모

## 한 줄 역할 정의
비서 자비스(`secretary`)는 앞으로 **국시 준비 플래너의 총괄 코디네이터/PM** 역할을 맡는다.

즉,
- 강렬이 플래너 관련 요청을 넣는 **주 창구**가 되고,
- 내부적으로는 **자비스 전공의(clerk)** 와 **자비스 교수님(academic)** 의견을 받아,
- 최종 일정안/우선순위안을 정리해서 강렬에게 보여주고,
- 실제 앱 수정이나 무거운 실행이 필요하면 **메인 자비스(main)** 에게 넘긴다.

---

## 왜 비서 자비스가 맡는가
현재 봇 역할 분담상:
- `secretary` = Gmail/Notion/옵시디언 감시, 중요 메일 요약, 일정 누락 체크, 옵시디언 일정 정리
- `clerk` = 실습 일정/현장 제약
- `academic` = 국시 전략/강의/문풀/복습 로직
- `main` = 코드/자동화/실행

플래너는 본질적으로
1. 일정 조율,
2. 우선순위 통합,
3. 사용자-facing 정리
가 핵심이므로, **비서 자비스가 조정자 역할을 맡는 것이 가장 자연스럽다.**

- `~/.openclaw/workspace/kmle-planner/DEPLOY_GITHUB.md`
- `~/.openclaw/workspace/kmle-planner/supabase/SETUP.md`

### 현재 앱 기능
1. 과목별 5단계 진행도 관리
   - 강의 듣기
   - 개념 틀 짜기
   - 문제 풀기
   - 오답노트/개념틀 수정
   - 복습하기
2. 과목별 학습량 입력 (`done / total`)
3. 오늘 일정 / 이번 주 일정 / 밀린 일정 표시
4. 복습·오답정리 우선 블록 표시
5. 취약 파트 / 다음 액션 / 과목 메모 저장
6. localStorage 저장
7. JSON export/import 지원
8. 예시 데이터 포함

### 실행 방법
가볍게는 브라우저에서 `index.html` 직접 열면 되고,
필요 시 메인 자비스가 로컬 서버(`python3 -m http.server 8765`)로 띄울 수 있다.

## 비서 자비스의 책임 범위
### 1) 사용자 창구
강렬이 플래너 관련 요청을 하면 **비서 자비스가 1차 응답 창구**가 된다.

예:
- 이번 주 플래너 짜줘
- 실습 반영해서 내일 일정 다시 짜줘
- 국시 우선순위 반영해서 복습 넣어줘
- 밀린 일정 기준으로 다시 정리해줘

### 2) 내부 조정
비서 자비스는 직접 모든 판단을 하지 않고,
필요 시 아래 두 봇에게 짧고 구조화된 질문을 던진다.

#### 자비스 전공의(clerk)에게 물을 것
- 이번 주 실습 고정 일정
- 공부 불가능 시간
- 이동/병원/회의 때문에 깨지는 시간 블록
- 피곤해서 가벼운 복습만 가능한 날
- 하루 현실 학습 가능량

#### 자비스 교수님(academic)에게 물을 것
- 이번 주 최우선 과목 3개
- 각 과목에서 지금 가장 밀린 단계
- 최소 복습 루프
- 취약 과목/파트 보정 우선순위
- 이번 주에 반드시 달성해야 할 최소 목표

### 3) 최종 통합
비서 자비스는 받은 정보를 이렇게 합친다.
- 전공의 정보 = **hard constraints**
- 교수님 정보 = **soft priorities**

즉,
- 전공의가 시간의 외곽 틀을 만들고
- 교수님이 그 안의 공부 전략을 채우며
- 비서 자비스가 강렬에게 보여줄 최종 플랜으로 정리한다.

### 4) 메인 자비스 호출 기준
아래는 비서 자비스가 직접 하지 말고 메인 자비스에게 넘긴다.
- 플래너 앱 코드 수정
- 자동화 기능 추가
- 데이터 구조 변경
- 긴 `exec` / 복잡한 로컬 실행
- approval이 뜰 수 있는 무거운 작업

## 운영 프로토콜
### A. 주간 계획 수립
1. 강렬이 비서 자비스에게 이번 주 플래너 요청
2. 비서 자비스가 전공의에게 실습 제약 확인
3. 비서 자비스가 교수님에게 국시 우선순위 확인
4. 비서 자비스가 둘을 합쳐 주간 계획안 제시
5. 필요 시 메인 자비스가 앱/데이터에 반영

### B. 일일 재조정
1. 실습 일정 변동 여부를 전공의에게 확인
2. 복습/취약과목 보정 필요 여부를 교수님에게 확인
3. 비서 자비스가 내일 일정 재배치안 작성

### C. 공부 후 업데이트
1. 강렬이 실제 수행량/밀린 일정 보고
2. 교수님이 진행도 및 복습 전략 재평가
3. 전공의가 현실 부담도 재판단
4. 비서 자비스가 수정된 플래너 제안

## 질문 방식 규칙
비서 자비스는 서브봇에게 질문할 때 다음을 지킨다.

### 좋은 질문 예시
- "이번 주 실습 때문에 절대 못 건드리는 시간만 bullet 5개로 줘."
- "이번 주 국시 최우선 과목 3개와 각 과목의 가장 밀린 단계만 줘."
- "내일 일정 기준으로 가벼운 복습만 가능한 시간대를 3개만 줘."

### 피해야 할 질문
- "전반적으로 보고 알아서 다 해줘"
- "전체적으로 다 정리해서 최적화해줘"

즉 **짧고 구조화된 질문**만 던진다.

## 출력 형식 권장
비서 자비스가 강렬에게 보여줄 때는 항상 아래 형식을 우선한다.

### 1. 요약
- 이번 주 핵심 포인트 3줄

### 2. 고정 제약
- 실습/병원 때문에 못 건드리는 시간

### 3. 국시 우선순위
- 이번 주 최우선 과목/단계

### 4. 최종 일정안
- 오늘 / 이번 주 / 복습 / 밀린 일정

### 5. 선택지
- 시간이 부족할 때 뺄 것 / 꼭 지킬 것

## 채팅방 운영 원칙
그룹채팅은 2단계 옵션이다.
처음부터 상시 그룹채팅으로 운영하지 않는다.

기본은:
- 강렬 ↔ 비서 자비스
- 비서 자비스 ↔ 전공의 / 교수님 / 메인 자비스

즉 **겉으로는 단일 창구, 내부는 협업** 구조를 유지한다.

## 한 줄 정리
비서 자비스는 앞으로 **국시 준비 플래너의 총괄 조정자**다.

이를 통합해 강렬에게 가장 쓰기 쉬운 일정안으로 정리해 보여주는 것이 핵심 임무다.
